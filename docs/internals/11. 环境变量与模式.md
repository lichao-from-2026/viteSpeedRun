## 11.1 环境变量基础

### 11.1.1 .env 文件

```bash
.env                # 所有环境都会加载
.env.local          # 所有环境，本地覆盖，应该被 git 忽略
.env.[mode]         # 特定模式
.env.[mode].local   # 特定模式的本地覆盖
```

### 11.1.2 环境变量规则

```env
# .env
# 必须以 VITE_ 开头才能在客户端代码中访问
VITE_APP_TITLE=My App
VITE_API_BASE=/api
```

```typescript
// main.ts
console.log(import.meta.env.VITE_APP_TITLE);
```

## 11.2 模式（Mode）

### 11.2.1 内置模式

| 命令 | 模式 | NODE_ENV |
|------|------|---------|
| `vite` | `development` | `development` |
| `vite build` | `production` | `production` |
| `vite preview` | `production` | `production` |

### 11.2.2 自定义模式

```bash
# 使用自定义模式构建
vite build --mode staging
```

## 11.3 import.meta.env

### 11.3.1 内置变量

```typescript
import.meta.env.MODE      // 当前模式
import.meta.env.BASE_URL  // 配置的 base
import.meta.env.PROD      // 是否生产环境
import.meta.env.DEV       // 是否开发环境
import.meta.env.SSR       // 是否 SSR 环境
```

### 11.3.2 使用环境变量

```typescript
// 开发环境检查
if (import.meta.env.DEV) {
  console.log('This is development');
}

// API 配置
const api = import.meta.env.VITE_API_BASE;
```

## 11.4 环境变量加载机制

### 11.4.1 加载顺序

```mermaid
flowchart TD
    A[开始加载环境变量] --> B[.env]
    B --> C[.env.local]
    C --> D[.env.[mode]]
    D --> E[.env.[mode].local]
    E --> F[合并所有变量]
```

### 11.4.2 优先级

```env
# .env（优先级最低）
VITE_APP_TITLE=My App
VITE_LOG_LEVEL=info

# .env.local（本地覆盖）
VITE_API_BASE=http://localhost:3000

# .env.development（开发模式）
VITE_API_BASE=http://localhost:3000
VITE_DEBUG=true

# .env.development.local（本地最高优先级）
VITE_API_BASE=http://localhost:8080
```

## 11.5 环境变量注入原理

### 11.5.1 Vite 如何注入

```typescript
// Vite 内部实现
const envVars = {
  'VITE_APP_TITLE': 'My App',
  'VITE_API_BASE': '/api',
};

// 替换代码中的 import.meta.env
code.replace('import.meta.env.VITE_APP_TITLE', JSON.stringify(envVars.VITE_APP_TITLE));
```

### 11.5.2 构建时注入

```typescript
// 源代码
if (import.meta.env.DEV) {
  console.log('Debug');
}

// 生产构建后
if (false) {
  console.log('Debug');
}
// 这部分代码会被 terser 完全移除！
```

## 11.6 实际使用示例

### 11.6.1 配置不同环境的 API 地址

```env
# .env.development
VITE_API_BASE=http://localhost:3000

# .env.staging
VITE_API_BASE=https://staging.example.com

# .env.production
VITE_API_BASE=https://api.example.com
```

```typescript
// 使用
const apiClient = axios.create({
  baseURL: import.meta.env.VITE_API_BASE,
});
```

### 11.6.2 特性开关

```env
# .env.development
VITE_FEATURE_DEBUG=true
VITE_FEATURE_PAYMENT=false
```

```typescript
if (import.meta.env.VITE_FEATURE_DEBUG) {
  console.log('Debug mode enabled');
}
```

## 11.7 实验：理解环境变量

### 实验 11.7.1：测试环境变量加载

1. 创建 `.env.development`
2. 添加环境变量
3. 运行 `npm run dev`
4. 验证变量是否生效

### 实验 11.7.2：测试构建时注入

1. 使用 `import.meta.env.PROD` 包裹代码
2. 运行构建
3. 查看输出代码是否被移除

## 11.8 常见问题

### 问题 1：环境变量没有生效？

**原因：** 没有以 VITE_ 开头

**解决方法：** 确保变量名以 VITE_ 开头

### 问题 2：多个环境变量冲突？

**原因：** 多个 .env 文件冲突

**解决方法：** 理解优先级，按正确顺序配置

### 问题 3：如何在 Node.js 中访问环境变量？

**方法：** 使用 `process.env`

```typescript
// vite.config.ts
console.log(process.env.VITE_APP_TITLE);
```

---

## 11.9 总结

环境变量与模式：

1. **.env 文件**：支持多个环境
2. **模式（Mode）**：区分开发、测试、生产等
3. **import.meta.env**：客户端访问环境变量
4. **变量前缀**：必须是 VITE_ 开头
5. **构建时注入**：代码会被替换和优化

理解环境变量有助于更好地管理项目配置！
