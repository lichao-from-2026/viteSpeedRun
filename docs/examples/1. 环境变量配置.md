# 环境变量配置

## 1. 默认规则

### 1.1 环境变量文件命名规则

Vite 支持以下环境变量文件：

- `.env` - 所有环境通用
- `.env.local` - 所有环境通用，但不会被 git 提交（需要添加到 .gitignore）
- `.env.development` - 开发环境专用
- `.env.production` - 生产环境专用
- `.env.development.local` - 开发环境专用，不会被 git 提交
- `.env.production.local` - 生产环境专用，不会被 git 提交

### 1.2 环境变量前缀

在 Vite 中，只有以 `VITE_` 开头的环境变量才会被暴露到浏览器环境中。

例如：
- `VITE_API_URL` - 会被暴露到浏览器环境
- `API_URL` - 不会被暴露到浏览器环境（仅在构建时可用）

### 1.3 环境模式

- 开发环境：`npm run dev` 使用 `.env.development`
- 生产环境：`npm run build` 使用 `.env.production`

## 2. 定义与使用

### 2.1 创建环境变量文件

#### 开发环境变量文件（.env.development）

```env
# 开发环境 API 地址
VITE_API_URL=http://localhost:3000/api

# 开发环境调试模式
VITE_DEBUG=true

# 开发环境特性开关
VITE_FEATURE_FLAG=true
```

#### 生产环境变量文件（.env.production）

```env
# 生产环境 API 地址
VITE_API_URL=https://api.example.com

# 生产环境调试模式
VITE_DEBUG=false

# 生产环境特性开关
VITE_FEATURE_FLAG=false
```

#### 通用环境变量文件（.env）

```env
# 通用环境变量
VITE_APP_NAME=Vite Speed Run
VITE_APP_VERSION=1.0.0
```

### 2.2 在代码中使用环境变量

在前端代码中，可以通过 `import.meta.env` 访问环境变量：

```typescript
// 访问环境变量
const apiUrl = import.meta.env.VITE_API_URL;
const debugMode = import.meta.env.VITE_DEBUG === 'true';
const appName = import.meta.env.VITE_APP_NAME;

// 其他内置环境变量
const isDevelopment = import.meta.env.DEV; // 开发环境为 true
const isProduction = import.meta.env.PROD; // 生产环境为 true
const baseUrl = import.meta.env.BASE_URL; // 项目基础路径
```

### 2.3 类型定义

在 TypeScript 项目中，建议在 `src/vite-env.d.ts` 文件中添加环境变量的类型定义：

```typescript
/// <reference types="vite/client" />

interface ImportMetaEnv {
  readonly VITE_API_URL: string;
  readonly VITE_DEBUG: string;
  readonly VITE_FEATURE_FLAG: string;
  readonly VITE_APP_NAME: string;
  readonly VITE_APP_VERSION: string;
  // 更多环境变量...
}

interface ImportMeta {
  readonly env: ImportMetaEnv;
}
```

## 3. 最佳实践

### 3.1 安全性

- 不要在环境变量中存储敏感信息，如 API 密钥、数据库密码等
- 使用 `.local` 文件存储本地开发配置，避免提交到版本控制系统
- 在生产环境中使用服务器环境变量，而不是文件

### 3.2 管理建议

- 创建 `.env.example` 文件作为模板，包含所有需要的环境变量
- 在团队协作中，确保所有成员都了解环境变量的配置要求
- 定期检查和更新环境变量配置

## 4. 示例代码

### 4.1 在组件中使用环境变量

```tsx
import React from 'react';

function App() {
  const apiUrl = import.meta.env.VITE_API_URL;
  const appName = import.meta.env.VITE_APP_NAME;
  const isDebug = import.meta.env.VITE_DEBUG === 'true';

  return (
    <div>
      <h1>{appName}</h1>
      <p>API 地址: {apiUrl}</p>
      <p>调试模式: {isDebug ? '开启' : '关闭'}</p>
    </div>
  );
}

export default App;
```

### 4.2 在 API 服务中使用环境变量

```typescript
// src/services/api.ts

const API_BASE_URL = import.meta.env.VITE_API_URL;

export async function fetchData(endpoint: string) {
  const response = await fetch(`${API_BASE_URL}${endpoint}`);
  return response.json();
}
```

## 5. 常见问题

### 5.1 环境变量不生效

- 检查环境变量文件是否正确命名
- 确保环境变量以 `VITE_` 开头
- 重启开发服务器
- 检查是否有语法错误

### 5.2 类型错误

- 确保在 `vite-env.d.ts` 中添加了环境变量的类型定义
- 检查 TypeScript 配置是否正确

### 5.3 生产环境配置

- 在部署前确保生产环境变量文件正确配置
- 考虑使用 CI/CD 工具管理生产环境变量
