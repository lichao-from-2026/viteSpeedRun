# 3. 多页面配置

***

## 文档概述

本文档专注于 Vite 多页面应用（Multi-Page Application, MPA）的配置，从基础搭建到高级技巧的完整指南。

**学习目标：**
- 理解多页面应用与单页面应用的区别

- 学会配置多个入口文件
- 了解多页面应用的构建和优化
- 实操搭建一个完整的多页面 Vite 项目

***

## 一、多页面应用简介

### 1.1 什么是多页面应用

多页面应用（MPA）是指一个应用包含多个 HTML 页面，每个页面有独立的入口文件。

**特点：**
- 每个页面有独立的 URL
- 页面跳转需要刷新浏览器
- SEO 友好
- 适合内容型网站

***

### 1.2 单页面 vs 多页面

| 特性 | 单页面应用（SPA） | 多页面应用（MPA） |
|------|-----------------|-----------------|
| 页面跳转 | 无刷新（前端路由） | 刷新浏览器 |
| SEO | 较差（需 SSR） | 优秀 |
| 首屏加载 | 较慢（需加载全部） | 较快（只加载当前页） |
| 开发复杂度 | 较高（路由管理） | 较低 |
| 适用场景 | 交互复杂的应用 | 内容型网站 |

### 2.4 配置 vite.config.ts

**最简单的配置（自动检测）：**

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()]
})
```

Vite 会自动检测根目录下的所有 `.html` 文件作为入口。

---

**显式配置（推荐）：**

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import { resolve } from 'path'

export default defineConfig({
  plugins: [react()],
  
  build: {
    rollupOptions: {
      input: {
        main: resolve(__dirname, 'index.html'),
        about: resolve(__dirname, 'about.html'),
        contact: resolve(__dirname, 'contact.html')
      }
    }
  }
})
```

### 3.4 自动检测所有页面

**使用 glob 自动检测：**

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import { resolve } from 'path'
import { glob } from 'glob'

export default defineConfig(async () => {
  // 自动查找所有 src/pages/*/index.html
  const pages = await glob('src/pages/*/index.html')
  
  // 构建 input 对象
  const input: Record<string, string> = {}
  pages.forEach((page) => {
    const name = page.split('/')[2] // 从路径中提取页面名
    input[name] = resolve(__dirname, page)
  })
  
  return {
    plugins: [react()],
    
    build: {
      rollupOptions: {
        input
      }
    }
  }
})
```

**需要安装 glob：**

```bash
npm install -D glob
```

### 4.1 代码分割

**分割公共代码：**

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

  plugins: [react()],

    rollupOptions: {
      input: {
        main: resolve(__dirname, 'index.html'),
        about: resolve(__dirname, 'about.html'),
        contact: resolve(__dirname, 'contact.html')

      output: {
        // 分割公共代码

  }
```

### 5.5 文章详情页

**src/pages/article/Article.tsx：**

```tsx
import React, { useEffect, useState } from 'react'
import { Header } from '../../components/Header'
import { Footer } from '../../components/Footer'
import { articles, Article } from '../../data/articles'

function ArticleDetail() {
  const [article, setArticle] = useState<Article | null>(null)

  useEffect(() => {
    const params = new URLSearchParams(window.location.search)
    const id = params.get('id')
    if (id) {
      const found = articles.find(a => a.id === parseInt(id))
      setArticle(found || null)
    }
  }, [])

  if (!article) {
    return (
      <div>
        <Header />
        <p>文章未找到</p>
        <Footer />
      </div>
    )
  }

  return (
    <div>
      <Header />
      <article>
        <h1>{article.title}</h1>
        <p>{article.date}</p>
        <div>{article.content}</div>
      </article>
      <a href="/articles.html">← 返回文章列表</a>
      <Footer />
    </div>
  )
}

export default ArticleDetail
```

## 六、最佳实践

### 6.1 项目组织

1. 页面集中管理
   - 所有页面放在 `src/pages/` 目录
   - 每个页面有独立的文件夹
   - 共享组件放在 `src/components/`

2. 入口文件命名
   - HTML 文件使用 `index.html`
   - JS/TS 入口使用 `main.ts`
   - 组件使用页面名命名

***

### 6.2 配置优化

1. 显式配置入口
   - 推荐在 `rollupOptions.input` 中显式配置
   - 避免自动检测的不确定性

2. 代码分割
   - 公共库单独打包
   - 共享组件单独打包
   - 减少重复加载

***

### 6.3 部署注意事项

1. URL 处理
   - 确保链接使用正确的 `.html` 后缀
   - 或配置服务器重写规则

2. Base URL
   - 部署到子路径时配置 `base`
   - 所有链接使用相对路径
