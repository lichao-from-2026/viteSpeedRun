## 一、tree-shaking 开启

### 1.1 什么是 tree-shaking

tree-shaking 是一种移除 JavaScript 中未使用代码的技术，能够显著减少最终打包文件的大小。

**原理：**

- 利用 ES Module 的静态结构（import/export）
- 编译时分析哪些代码被使用，哪些没有被使用
- 删除未使用的代码

***

### 1.2 Vite 中的 tree-shaking

Vite 默认已经开启了 tree-shaking，但我们可以进行一些优化配置。

**vite.config.ts 配置：**

```typescript
import { defineConfig } from 'vite'

export default defineConfig({
  build: {
    // 开启 minify（默认开启）
    minify: 'terser',
    
    // terser 配置
    terserOptions: {
      compress: {
        // 删除 console
        drop_console: true,
        // 删除 debugger
        drop_debugger: true
      }
    },
    
    // 确保使用 ES Module
    rollupOptions: {
      output: {
        // 使用 ES Module 格式
        format: 'es'
      }
    }
  }
})
```

***

### 1.3 package.json 配置

确保在 package.json 中正确配置：

```json
{
  "type": "module",
  "sideEffects": false
}
```

**sideEffects 说明：**

- `false` - 所有文件都没有副作用，可以安全地 tree-shake
- 数组 - 指定有副作用的文件，例如 `["*.css"]`

## 二、代码分割

### 1. 动态导入：`import()` 函数

**功能：** 不一次性加载所有 JS，分割成独立 chunk，用到时再加载。

**原理：**

- 静态导入：在 编译时 就加载所有模块
- 动态导入：在 运行时 按需加载模块
- 每个动态导入的模块会被单独打包成一个 chunk

```TypeScript
// 静态导入（打包在一起）
import Home from './Home'

// 动态导入（自动分割成独立文件）
const Home = React.lazy(() => import('./Home'))
```

***

### 2. 手动分割：自己写 `import()` 手动分割

**React 案例：**

```typescript
import { lazy, Suspense } from 'react'
import { Routes, Route } from 'react-router-dom'

// 手动写 import() 进行分割
const Home = lazy(() => import('./pages/Home'))
const About = lazy(() => import('./pages/About'))
const Dashboard = lazy(() => import('./pages/Dashboard'))

function App() {
  return (
    <Suspense fallback={<div>加载中...</div>}>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/dashboard" element={<Dashboard />} />
      </Routes>
    </Suspense>
  )
}
```

**优点：**

- 按需加载（访问哪个路由才加载对应代码）
- 首屏加载更快（只加载当前页面代码）
- 提供加载状态和错误处理

***

### 3. Vite 配置：按路由分割 / 按库分割

**功能：** Vite 帮你自动批量写动态导入，不用你手动写。

**按库分割：**

```typescript
import { defineConfig } from 'vite'

export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          // 按库分割
          'vendor-react': ['react', 'react-dom'],
          'vendor-ui': ['antd', '@ant-design/icons'],
          'vendor-utils': ['lodash', 'dayjs', 'axios']
        }
      }
    }
  }
})
```

**按路由分割（函数形式）：**

```typescript
import { defineConfig } from 'vite'

export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks(id) {
          // 按路由分割
          if (id.includes('src/pages/Home')) {
            return 'page-home'
          }
          if (id.includes('src/pages/About')) {
            return 'page-about'
          }
          
          // 按库分割
          if (id.includes('node_modules')) {
            return 'vendor'
          }
        }
      }
    }
  }
})
```

**优点：**

- Vite 自动批量处理，不用手动写每个 import()
- 灵活控制哪些模块打包在一起
- 优化 chunk 大小和缓存效率

***

### 4. 三种方式对比

| 方式                      | 说明                               | 适用场景          |
| ----------------------- | -------------------------------- | ------------- |
| **动态导入** **`import()`** | 基础技术，手动写 import()                | 简单场景，少量路由     |
| **手动分割**                | 自己写 React.lazy + import()        | React 应用，按需加载 |
| **Vite 配置**             | vite.config.ts 配置 `manualChunks` | 大型应用，批量处理     |

**推荐配合使用：**

- Vite 配置：按库分割第三方库
- 手动分割：按路由分割页面代码
- 两者结合：最佳性能优化方案

## 三、压缩 JS/CSS

### 3.1 JS 压缩

Vite 内置了 terser 用于压缩 JavaScript。

**配置示例：**

```typescript
import { defineConfig } from 'vite'

export default defineConfig({
  build: {
    // 使用 terser 压缩
    minify: 'terser',
    
    // terser 高级配置
    terserOptions: {
      compress: {
        // 删除 console
        drop_console: true,
        // 删除 debugger
        drop_debugger: true,
        // 移除未使用的变量
        unused: true,
        // 内联简单函数
        inline: true
      },
      format: {
        // 删除注释
        comments: false
      }
    }
  }
})
```

***

### 3.2 CSS 压缩

Vite 会自动压缩 CSS，我们可以进行额外配置。

**使用 postcss 插件：**

```typescript
import { defineConfig } from 'vite'

export default defineConfig({
  css: {
    postcss: {
      plugins: [
        // 自动添加浏览器前缀
        require('autoprefixer'),
        // CSS 压缩
        require('cssnano')({
          preset: 'default'
        })
      ]
    }
  }
})
```

## 四、图片压缩与懒加载

### 4.1 图片压缩

**使用 vite-plugin-imagemin 插件：**

```bash
npm install -D vite-plugin-imagemin
```

**vite.config.ts 配置：**

```typescript
import { defineConfig } from 'vite'
import viteImagemin from 'vite-plugin-imagemin'

export default defineConfig({
  plugins: [
    viteImagemin({
      gifsicle: {
        optimizationLevel: 7, // GIF 优化级别（1-7），7 最高
        interlaced: false // 是否启用隔行扫描
      },
      optipng: {
        optimizationLevel: 7 // PNG 优化级别（0-7），7 最高
      },
      mozjpeg: {
        quality: 80 // JPEG 质量（0-100），80 是平衡质量和大小的推荐值
      },
      pngquant: {
        quality: [0.8, 0.9], // PNG 质量范围
        speed: 4 // 压缩速度（1-11），1 最慢但质量最好，11 最快但质量较差
      },
      svgo: {
        plugins: [
          {
            name: 'removeViewBox' // 移除 SVG 的 viewBox 属性
          }
        ]
      }
    })
  ]
})
```

**构建输出效果：**

- **图片大小显著减少** - JPEG/PNG 图片可减少 50-80%，SVG 可减少 30-60%
- **质量保持** - 压缩后图片质量基本无肉眼可见差异
- **自动处理** - 构建过程中自动压缩所有图片
- **支持多种格式** - 支持 JPEG、PNG、GIF、SVG 等格式
- **输出目录**：
  ```
  dist/
  ├── assets/
  │   ├── images/
  │   │   ├── logo-abc123.png      # 压缩后的 PNG
  │   │   ├── banner-def456.jpg    # 压缩后的 JPEG
  │   │   └── icon-ghi789.svg      # 压缩后的 SVG
  │   └── ...
  └── ...
  ```

### 4.2 图片懒加载

**React 中使用 Intersection Observer：**

```typescript
import { useRef, useEffect, useState } from 'react'

// 图片出现在视口时，才会加载。
function LazyImage({ src, alt }) {
  const [imageSrc, setImageSrc] = useState('')
  const imgRef = useRef<HTMLImageElement>(null)

  useEffect(() => {
    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          setImageSrc(src)
          observer.unobserve(entry.target)
        }
      }
    )

    if (imgRef.current) {
      observer.observe(imgRef.current)
    }

    return () => {
      if (imgRef.current) {
        observer.unobserve(imgRef.current)
      }
    }
  }, [src])

  return <img ref={imgRef} src={imageSrc} alt={alt} />
}

// 使用
<LazyImage src="large-image.jpg" alt="Description" />
```

## 五、完整配置示例

### 5.1 最佳实践配置

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import viteImagemin from 'vite-plugin-imagemin'

export default defineConfig({
  plugins: [
    react(),
    // 图片压缩
    viteImagemin({
      gifsicle: { optimizationLevel: 7 },
      optipng: { optimizationLevel: 7 },
      mozjpeg: { quality: 80 },
      pngquant: { quality: [0.8, 0.9] }
    })
  ],
  
  build: {
    // JS 压缩
    minify: 'terser',
    terserOptions: {
      compress: {
        drop_console: true,
        drop_debugger: true
      }
    },
    
    // 代码分割
    rollupOptions: {
      output: {
        manualChunks: {
          'vendor-react': ['react', 'react-dom'],
          'vendor-ui': ['antd', '@ant-design/icons'],
          'vendor-utils': ['lodash', 'dayjs', 'axios']
        }
      }
    },
    
    // 生成 sourcemap（生产环境可以关闭）
    sourcemap: false,
    
    // 清空输出目录
    emptyOutDir: true
  },
  
  // CSS 压缩
  css: {
    postcss: {
      plugins: [
        require('autoprefixer'),
        require('cssnano')({
          preset: 'default'
        })
      ]
    }
  }
})
```

**构建输出效果：**

- **文件结构清晰** - 代码和资源文件分离
- **体积显著减少** - 经过压缩和分割后，整体体积可减少 60-80%
- **按需加载** - 不同路由和功能模块独立加载
- **缓存优化** - 第三方库和共享代码独立缓存
- **加载速度快** - 首屏加载时间大幅减少

**最终输出目录：**
