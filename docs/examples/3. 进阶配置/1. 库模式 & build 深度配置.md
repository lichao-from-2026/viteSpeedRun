## 一、库模式简介

### 1.1 什么是库模式

库模式是 Vite 专门为构建可复用库（如组件库、工具库、npm 包）提供的构建模式。

**普通构建 vs 库模式对比：**

| 特性   | 普通应用构建    | 库模式构建            |
| ---- | --------- | ---------------- |
| 目标   | 可直接运行的应用  | 可复用的库            |
| 打包依赖 | 打包所有依赖    | 不打包外部依赖          |
| 入口文件 | HTML 文件   | JS/TS 文件         |
| 输出格式 | 单一格式      | 多种格式（es/cjs/umd） |
| 使用方式 | 浏览器直接打开   | 其他项目 npm install |
| 用途   | 网站、Web 应用 | 组件库、工具库、npm 包    |

***

### 1.2 最简单的库模式案例

让我们从一个最简单的配置开始：

```typescript
import { defineConfig } from 'vite'

export default defineConfig({
  build: {
    lib: {
      entry: './src/index.ts',
      name: 'MyLibrary',
      fileName: 'my-library',
      formats: ['es', 'cjs']
    }
  }
})
```

**文件结构：**

```
my-library/
├── src/
│   └── index.ts
├── vite.config.ts
└── package.json
```

**src/index.ts：**

```typescript
export function hello(name: string) {
  return `Hello, ${name}!`
}
```

**构建后输出：**

```
dist/
├── my-library.js    (ES Module)
└── my-library.cjs  (CommonJS)
```

***

### 1.3 配合 rollupOptions 的简单案例

当需要更精细控制时，可以配合 `rollupOptions`：

```typescript
import { defineConfig } from 'vite'

export default defineConfig({
  build: {
    lib: {
      entry: './src/index.ts',
      name: 'MyUtils',
      fileName: (format) => `my-utils.${format}.js`,
      formats: ['es', 'cjs', 'umd']
    },
    rollupOptions: {
      // 外部化依赖，不打包到库中
      external: ['lodash', 'dayjs'],
      output: {
        // UMD 格式的全局变量名
        globals: {
          lodash: '_',
          dayjs: 'dayjs'
        }
      }
    }
  }
})
```

**这个配置的作用：**

- `external` - 告诉 Vite 不要把 lodash 和 dayjs 打包进去
- `globals` - 告诉 UMD 格式这些库的全局变量名是什么

### 2.1 库编写

#### 2.1.1 创建项目结构

一个标准的库项目结构如下：

```
my-awesome-library/
├── src/
│   ├── index.ts          # 入口文件
│   ├── utils/            # 工具函数
│   │   └── format.ts
│   └── components/       # 组件（如果是组件库）
│       └── Button/
│           ├── Button.tsx
│           └── Button.css
├── vite.config.ts        # Vite配置
├── tsconfig.json         # TypeScript配置
├── package.json          # 包配置
└── README.md             # 说明文档
```

***

#### 2.1.2 编写入口文件

入口文件（通常是 `src/index.ts`）负责导出库的所有内容：

```typescript
// 导出函数
export function hello(name: string) {
  return `Hello, ${name}!`
}

// 导出组件
export { Button } from './components/Button'
export type { ButtonProps } from './components/Button'

// 导出工具
export * from './utils/format'
```

***

#### 2.1.3 编写功能代码

**工具函数示例（src/utils/format.ts）：**

```typescript
export function capitalize(str: string) {
  return str.charAt(0).toUpperCase() + str.slice(1)
}

export function toLowerCase(str: string) {
  return str.toLowerCase()
}
```

**组件示例（src/components/Button/Button.tsx）：**

```typescript
import React from 'react'

export interface ButtonProps {
  children: React.ReactNode
  onClick?: () => void
}

export function Button({ children, onClick }: ButtonProps) {
  return (
    <button onClick={onClick}>
      {children}
    </button>
  )
}
```

### 2.2 发布准备

#### 2.2.1 完善 package.json

```json
{
  "name": "my-awesome-library",
  "version": "1.0.0",
  "description": "一个很棒的工具库",
  "author": "Your Name",
  "license": "MIT",
  "type": "module",
  
  "main": "./dist/my-awesome-library.cjs.js",
  "module": "./dist/my-awesome-library.js",
  "types": "./dist/index.d.ts",
  
  "exports": {
    ".": {
      "import": "./dist/my-awesome-library.js",
      "require": "./dist/my-awesome-library.cjs.js",
      "types": "./dist/index.d.ts"
    }
  },
  
  "files": ["dist", "README.md"],
  
  "peerDependencies": {
    "react": ">=17",
    "react-dom": ">=17"
  },
  
  "devDependencies": {
    "vite": "^5.0.0",
    "typescript": "^5.0.0",
    "vite-plugin-dts": "^3.0.0"
  },
  
  "scripts": {
    "build": "vite build",
    "prepublishOnly": "npm run build"
  }

```

**关键字段说明：**

| 字段                 | 说明              | 重要性 |
| ------------------ | --------------- | --- |
| `name`             | 包名（必须唯一）        | ⭐⭐⭐ |
| `version`          | 版本号（语义化）        | ⭐⭐⭐ |
| `main`             | CommonJS 入口     | ⭐⭐  |
| `module`           | ES Module 入口    | ⭐⭐  |
| `types`            | TypeScript 类型声明 | ⭐⭐⭐ |
| `exports`          | 现代导出配置（推荐）      | ⭐⭐⭐ |
| `files`            | 发布的文件列表         | ⭐⭐  |
| `peerDependencies` | 对等依赖            | ⭐⭐⭐ |

***

#### 2.2.2 配置 Vite

创建 `vite.config.ts` 文件：

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import dts from 'vite-plugin-dts'
import path from 'path'

export default defineConfig({
  plugins: [
    react(),
    dts() // 自动生成类型声明
  ],
  
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src')
    }
  },
  
  build: {
    lib: {
      entry: './src/index.ts',
      name: 'MyAwesomeLibrary',
      fileName: (format) => `my-awesome-library.${format}.js`,
      formats: ['es', 'cjs', 'umd']
    },
    
    rollupOptions: {
      external: ['react', 'react-dom'],
      output: {
        globals: {
          react: 'React',
          'react-dom': 'ReactDOM'
        }
      }
    }
  }
})
```

***

#### 2.2.3 配置 TypeScript

创建或修改 `tsconfig.json`：

```json
{
  "compilerOptions": {
    "target": "ES2020", // 目标 ECMAScript 版本
    "useDefineForClassFields": true, // 使用 Object.defineProperty 定义类字段
    "lib": ["ES2020", "DOM", "DOM.Iterable"], // 编译时使用的库
    "module": "ESNext", // 模块系统
    "skipLibCheck": true, // 跳过类型检查库文件
    "moduleResolution": "bundler", // 模块解析策略
    "allowImportingTsExtensions": true, // 允许导入 .ts 扩展名
    "resolveJsonModule": true, // 允许导入 JSON 文件
    "isolatedModules": true, // 每个文件作为单独模块
    "noEmit": true, // 不生成输出文件
    "jsx": "react-jsx", // JSX 编译方式
    "strict": true, // 启用所有严格类型检查
    "noUnusedLocals": true, // 禁止未使用的局部变量
    "noUnusedParameters": true, // 禁止未使用的参数
    "noFallthroughCasesInSwitch": true, // 禁止 switch 语句穿透
    "declaration": true, // 生成类型声明文件
    "declarationDir": "./dist" // 类型声明文件输出目录
  },
  "include": ["src"], // 包含的文件
  "references": [{ "path": "./tsconfig.node.json" }] // 引用其他配置
}
```

### 2.4 发布包

#### 2.4.1 发布到 npm（公开包）

**完整步骤：**

1. 注册 npm 账号
   - 访问 <https://www.npmjs.com/signup>
   - 创建账号并验证邮箱
2. 登录 npm
   ```bash
   npm login
   ```
   按提示输入用户名、密码和邮箱
3. 检查包名是否可用
   ```bash
   npm view my-awesome-library
   ```
   如果报错说明包名已被占用，需要换个名字
4. 发布前预览
   ```bash
   npm publish --dry-run
   ```
   这会模拟发布，但不会真的发布，可以检查是否有问题
5. 正式发布
   ```bash
   npm publish
   ```

### 2.6 版本管理

#### 2.6.1 语义化版本

| 版本格式                | 说明           | 示例      |
| ------------------- | ------------ | ------- |
| `MAJOR.MINOR.PATCH` | 主版本.次版本.补丁版本 | `1.2.3` |

**版本号递增规则：**

- **MAJOR（主版本）** - 不兼容的 API 修改
- **MINOR（次版本）** - 向下兼容的功能性新增
- **PATCH（补丁版本）** - 向下兼容的问题修正

**使用 npm 命令更新版本：**

```bash
# 补丁版本（1.0.0 → 1.0.1）
npm version patch

# 次要版本（1.0.0 → 1.1.0）
npm version minor

# 主要版本（1.0.0 → 2.0.0）
npm version major
```

## 三、build.lib 深度配置

### 3.1 配置项详解

| 配置项        | 类型                 | 必填 | 说明                   |
| ---------- | ------------------ | -- | -------------------- |
| `entry`    | string             | ✅  | 库的入口文件路径             |
| `name`     | string             | ⭕  | 库的全局变量名（UMD/IIFE 使用） |
| `fileName` | string \| function | ⭕  | 输出文件名                |
| `formats`  | string\[]          | ⭕  | 输出格式数组               |

***

### 3.2 entry - 入口文件

**最简单用法：**

```typescript
lib: {
  entry: './src/index.ts'
}
```

**支持的文件类型：**

- `.ts` - TypeScript
- `.tsx` - TypeScript + JSX
- `.js` - JavaScript
- `.jsx` - JavaScript + JSX

***

### 3.3 name - 全局变量名

**什么时候需要？**

- 输出 UMD 格式时
- 输出 IIFE 格式时

**示例：**

```typescript
lib: {
  entry: './src/index.ts',
  name: 'MyUtils',
  formats: ['umd']
}
```

**使用 UMD 格式：**

```html
<script src="dist/my-utils.umd.js"></script>
<script>
  // 使用全局变量 MyUtils
  console.log(MyUtils.hello('World'))
</script>
```

***

### 3.4 fileName - 输出文件名

**用法 1：字符串（简单）**

```typescript
lib: {
  entry: './src/index.ts',
  fileName: 'my-library'
}
```

输出：

```
dist/
├── my-library.js
├── my-library.cjs
└── my-library.umd.js
```

**用法 2：函数（灵活）**

```typescript
lib: {
  entry: './src/index.ts',
  fileName: (format) => `my-library.${format}.js`
}
```

输出：

```
dist/
├── my-library.es.js
├── my-library.cjs.js
└── my-library.umd.js
```

***

### 3.5 formats - 输出格式

**支持的格式：**

| 格式       | 说明        | 适用场景              |
| -------- | --------- | ----------------- |
| `'es'`   | ES Module | 现代浏览器、Node.js 14+ |
| `'cjs'`  | CommonJS  | Node.js 旧版本       |
| `'umd'`  | UMD       | 浏览器全局变量、AMD       |
| `'iife'` | IIFE      | 浏览器直接使用           |

**示例 1：只输出 ES 和 CommonJS（推荐）**

```typescript
lib: {
  entry: './src/index.ts',
  formats: ['es', 'cjs']
}
```

**示例 2：输出所有格式**

```typescript
lib: {
  entry: './src/index.ts',
  formats: ['es', 'cjs', 'umd', 'iife']
}
```

***

### 3.6 完整配置示例

```typescript
import { defineConfig } from 'vite'

export default defineConfig({
  build: {
    lib: {
      entry: './src/index.ts',
      name: 'MyLibrary',
      fileName: (format) => `my-library.${format}.js`,
      formats: ['es', 'cjs', 'umd']
    },
    
    // 输出目录
    outDir: 'dist',
    
    // 生成 sourcemap
    sourcemap: true,
    
    // 清空输出目录
    emptyOutDir: true
  }
})
```

## 四、build.rollupOptions 深度配置

### 4.1 简介

`rollupOptions` 允许直接配置底层的 Rollup 打包工具，提供更精细的控制。

**常用场景：**

- 外部化依赖
- 代码分割
- 自定义输出格式
- 使用 Rollup 插件

***

### 4.2 库构建中的 rollupOptions

#### 4.2.1 external - 外部化依赖

**作用：** 指定哪些依赖不应该被打包到库中。

**为什么需要？**

- 减少库的体积
- 避免依赖重复打包
- 让使用者自己管理依赖版本

**示例：**

```typescript
build: {
  lib: {
    entry: './src/index.ts',
    name: 'MyUI',
    formats: ['es', 'cjs', 'umd']
  },
  rollupOptions: {
    external: ['react', 'react-dom', 'lodash']
  }
}
```

**使用正则表达式：**

```typescript
rollupOptions: {
  external: [
    'react',
    'react-dom',
    /^lodash/,      // 所有 lodash 开头的包
    /^@types\//     // 所有类型声明包
  ]
}
```

***

#### 4.2.2 output.globals - 全局变量名

**作用：** 指定 external 依赖在 UMD/IIFE 格式中的全局变量名。

**示例：**

```typescript
build: {
  lib: {
    entry: './src/index.ts',
    name: 'MyUI',
    formats: ['umd']
  },
  rollupOptions: {
    external: ['react', 'react-dom'],
    output: {
      globals: {
        react: 'React',
        'react-dom': 'ReactDOM'
      }
    }
  }
}
```

**使用 UMD 格式：**

```html
<script src="https://cdn.jsdelivr.net/npm/react@18/umd/react.production.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/react-dom@18/umd/react-dom.production.min.js"></script>
<script src="dist/my-ui.umd.js"></script>
<script>
  // React 和 ReactDOM 已经是全局变量
  // MyUI 可以直接使用它们
</script>
```

***

#### 4.2.3 output.manualChunks - 代码分割

> 详情查看 docs/examples/3. 进阶配置/2. **优化打包体积**.md 的 \*\*代码分割 \*\*章节。

**作用：** 手动控制哪些模块应该打包到同一个 chunk 中。

**示例：**

```typescript
build: {
  lib: {
    entry: './src/index.ts',
    name: 'MyLibrary'
  },
  rollupOptions: {
    output: {
      manualChunks: {
        'vendor': ['lodash', 'dayjs'],
        'core': ['src/core'],
        'utils': ['src/utils']
      }
    }
  }
}
```

***

### 4.3 应用构建中的 rollupOptions

虽然本文档聚焦于库模式，但也了解一下应用构建中的用法。

#### 4.3.1 output.manualChunks - 按库分割

> 详情查看 docs/examples/3. 进阶配置/2. **优化打包体积**.md 的 \*\*代码分割 \*\*章节。

```typescript
build: {
  rollupOptions: {
    output: {
      manualChunks: {
        'vendor-react': ['react', 'react-dom'],
        'vendor-ui': ['antd', '@ant-design/icons'],
        'vendor-utils': ['lodash', 'dayjs', 'axios']
      }
    }
  }
}
```

***

#### 4.3.2 output.manualChunks - 按路由分割【重点 - 函数形式】

> 详情查看 docs/examples/3. 进阶配置/2. **优化打包体积**.md 的 \*\*代码分割 \*\*章节。

**作用：** 函数形式的 `manualChunks` 提供了更灵活的代码分割策略，可以根据模块的路径、名称等信息进行更精细的控制。

**优势：**

- 可以基于模块路径进行条件判断
- 支持更复杂的分割逻辑
- 可以同时处理路由分割和第三方库分割
- 适合大型应用的复杂代码分割需求

**构建输出效果：**

**1. 生成独立的路由 Chunk**

- 每个路由会被打包成单独的 JavaScript 文件
- 例如：`page-home-abc123.js`、`page-about-def456.js`、`page-dashboard-ghi789.js`
- 第三方库会被打包成共享的 `vendor-xxx.js`

**2. 加载行为效果**

- **首屏快** - 首次只加载当前路由代码，避免一次性加载所有路由
- **按需加载** - 导航到新路由时才加载对应代码
- **加载顺序优化** - 首次仅加载主包、当前路由 chunk、共享依赖，此后仅按序加载路由 chunk
- **更好的缓存利用** - 路由代码独立缓存，共享依赖只加载一次

**3. 构建输出结构**

```
dist/
├── assets/
│   ├── index-abc123.js          # 主 bundle
│   ├── page-home-def456.js       # 首页 chunk
│   ├── page-about-ghi789.js      # 关于页 chunk
│   ├── page-dashboard-jkl012.js  # 仪表盘 chunk
│   ├── vendor-mno345.js          # 第三方库
│   └── shared-pqr678.js          # 共享组件
└── index.html
```

#### 4.3.3 自定义 Chunk 输出格式

```typescript
build: {
  rollupOptions: {
    output: {
      // Chunk 文件名
      chunkFileNames: 'js/[name]-[hash].js',
      
      // 入口文件名
      entryFileNames: 'js/[name]-[hash].js',
      
      // 资源文件名
      assetFileNames: 'assets/[name]-[hash][extname]'
    }
  }
}
```

**文件名占位符：**

| 占位符         | 说明         |
| ----------- | ---------- |
| `[name]`    | 模块名称       |
| `[hash]`    | 内容哈希（用于缓存） |
| `[extname]` | 文件扩展名      |
| `[ext]`     | 文件扩展名（不带点） |

## 六、最佳实践

### 6.1 库模式最佳实践

1. TypeScript 支持
   - 确保 `tsconfig.json` 配置正确
   - 使用 `vite-plugin-dts` 自动生成类型声明
   - 在 package.json 中配置 `types` 字段
2. 依赖管理
   - 库的依赖放在 `peerDependencies`
   - 开发依赖放在 `devDependencies`
   - 避免过度使用 `dependencies`
3. 输出格式
   - 推荐同时输出 `es` 和 `cjs` 格式
   - 如果需要支持浏览器全局变量，再加 `umd`
   - 在 package.json 中配置 `exports` 字段
4. 文档完善
   - 提供清晰的 README
   - 说明安装和使用方法
   - 提供示例代码
   - 记录 API 文档

***

### 6.2 rollupOptions 最佳实践

1. 合理使用 external
   - 只 externalize 大型、稳定的库
   - 不要过度 externalize，影响开发体验
   - 配合 CDN 加载时确保版本一致
2. *代码分割策略【重点】*
   - 按路由分割，实现按需加载
   - 按库分割，利用浏览器缓存
   - 避免过度分割，增加请求数
3. 文件名设计
   - 使用 `[name]` 便于调试
   - 使用 `[hash]` 用于缓存控制
   - 合理组织目录结构
