# 1. 插件入门

## 文档概述

本文档将带你深入学习 Vite 插件的基础知识，从插件的作用、常用插件的使用，到自定义插件的开发和第三方插件的集成。

**学习目标：**

- 理解 Vite 插件的作用
- 掌握常用插件的安装与使用
- 学习自定义插件的开发
- 掌握第三方插件的集成方法

## 一、插件的作用

### 1.1 什么是 Vite 插件

Vite 插件是基于 Rollup 插件接口的扩展，同时提供了额外的 Vite 特有钩子。插件可以：

- 扩展 Vite 的核心功能
- 自定义构建行为
- 处理特殊文件类型
- 集成第三方工具
- 优化开发体验

### 1.2 插件的作用场景

常见的插件使用场景包括：

- **文件转换**：将 TS 转换为 JS，处理 CSS 预处理器等
- **资源处理**：处理图片、字体等静态资源
- **功能扩展**：添加 PWA 支持、SSR 支持等
- **开发工具**：热更新、代理配置、Mock 数据等
- **性能优化**：代码分割、压缩优化等

## 二、常用插件的安装与使用

### 2.1 官方插件

Vite 官方提供了一些常用插件：

#### 2.1.1 @vitejs/plugin-react

**作用**：React 项目的必备插件，提供 React Fast Refresh 支持。

**安装**：

```bash
npm install @vitejs/plugin-react -D
```

**使用**：

```typescript
// vite.config.ts
import { defineConfig } from 'vite'
// 引入 React 插件
import react from '@vitejs/plugin-react'

export default defineConfig({
  // 插件配置
  plugins: [
    // 使用 React 插件
    react()
  ]
})
```

#### 2.1.2 @vitejs/plugin-vue

**作用**：Vue 3 项目的必备插件，提供 Vue 组件支持。

**安装**：

```bash
npm install @vitejs/plugin-vue -D
```

**使用**：

```typescript
import { defineConfig } from 'vite'
// 引入 Vue 插件
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()]
})
```

### 2.2 常用社区插件

#### 2.2.1 vite-plugin-compression

**作用**：构建时生成压缩文件（gzip 或 brotli）。

**安装**：

```bash
npm install vite-plugin-compression -D
```

**使用**：

```typescript
import { defineConfig } from 'vite'
// 引入压缩插件
import compression from 'vite-plugin-compression'

export default defineConfig({
  plugins: [
    // 使用压缩插件
    compression({
      // 压缩算法：gzip 或 brotli
      algorithm: 'gzip',
      // 压缩后的文件扩展名
      ext: '.gz'
    })
  ]
})
```

#### 2.2.2 vite-plugin-svg-icons

**作用**：快速加载 SVG 图标。

**安装**：

```bash
npm install vite-plugin-svg-icons -D
```

**使用**：

```typescript
import { defineConfig } from 'vite'
// 引入 SVG 图标插件
import { createSvgIconsPlugin } from 'vite-plugin-svg-icons'
import path from 'path'

export default defineConfig({
  plugins: [
    // 使用 SVG 图标插件
    createSvgIconsPlugin({
      // SVG 图标文件夹路径
      iconDirs: [path.resolve(process.cwd(), 'src/assets/icons')],
      // 图标 ID 格式
      symbolId: 'icon-[dir]-[name]'
    })
  ]
})
```

#### 2.2.3 vite-plugin-mock

**作用**：提供 Mock 数据服务。

**安装**：

```bash
npm install vite-plugin-mock -D
```

**使用**：

```typescript
import { defineConfig } from 'vite'
// 引入 Mock 插件
import { viteMockServe } from 'vite-plugin-mock'

export default defineConfig({
  plugins: [
    // 使用 Mock 插件
    viteMockServe({
      // Mock 文件位置
      mockPath: 'mock',
      // 开发环境启用
      enable: true
    })
  ]
})
```

## 三、自定义插件示例

### 3.1 示例一：给代码添加版权注释

让我们创建一个简单的插件，自动给代码添加版权注释。

**插件代码**：

```typescript
// vite-plugin-copyright.ts
export function copyrightPlugin() {
  // 当前年份
  const year = new Date().getFullYear()
  // 版权注释
  const copyrightNotice = `/**
 * Copyright © ${year} My Company
 * All rights reserved.
 */\n`
  
  return {
    // 插件名称
    name: 'vite-plugin-copyright',
    
    // transform 钩子
    transform(code, id) {
      // 只处理 .js 和 .ts 文件
      if (id.match(/\.(js|ts)$/)) {
        // 返回添加版权注释后的代码
        return {
          code: copyrightNotice + code,
          map: null
        }
      }
      return null
    }
  }
}
```

**在 vite.config.ts 中使用**：

```typescript
import { defineConfig } from 'vite'
// 引入版权插件
import { copyrightPlugin } from './vite-plugin-copyright'

export default defineConfig({
  plugins: [
    // 使用版权插件
    copyrightPlugin()
  ]
})
```

### 3.2 示例二：处理自定义文件后缀

现在让我们创建一个处理自定义文件后缀的插件。

**插件代码**：

```typescript
// vite-plugin-yaml.ts
export function yamlPlugin() {
  return {
    // 插件名称
    name: 'vite-plugin-yaml',
    
    // 处理 YAML 文件转换
    transform(code, id) {
      // 只处理 .yaml 或 .yml 文件
      if (id.match(/\.(yaml|yml)$/)) {
        try {
          // 简单的 YAML 解析（实际项目可使用 js-yaml）
          const data = parseYaml(code)
          // 转换为 ES 模块
          return {
            code: `export default ${JSON.stringify(data)}`,
            map: null
          }
        } catch (e) {
          console.error('YAML 解析错误:', e)
          return null
        }
      }
      return null
    }
  }
}

// 简单的 YAML 解析函数
function parseYaml(code: string) {
  // 这是一个简化的实现
  // 实际项目中应使用 js-yaml 等库
  const result: Record<string, any> = {}
  code.split('\n').forEach(line => {
    const match = line.match(/(\w+):\s*(.+)/)
    if (match) {
      result[match[1]] = match[2]
    }
  })
  return result
}
```

**使用示例**：

```yaml
# config.yaml
title: My App
version: 1.0.0
```

```typescript
// 在代码中使用
import config from './config.yaml'

console.log(config.title) // "My App"
```

## 四、第三方插件集成【易错】

### 4.1 插件的配置顺序

插件的配置顺序非常重要，不同类型的插件需要按特定顺序加载，否则可能导致功能失效或冲突。

#### 推荐的插件配置顺序原则：

```
1. 框架插件（最先加载）
   ↓
2. 路径别名、模块解析插件
   ↓
3. 文件转换插件（如 TS、CSS 预处理器）
   ↓
4. 开发工具插件（Mock、代理等）
   ↓
5. 资源处理插件（图片、图标等）
   ↓
6. 构建优化插件（最后加载）
```

#### 完整的插件配置示例：

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import vue from '@vitejs/plugin-vue'
import path from 'path'
import vueJsx from '@vitejs/plugin-vue-jsx'
import { createSvgIconsPlugin } from 'vite-plugin-svg-icons'
import { viteMockServe } from 'vite-plugin-mock'
import viteCompression from 'vite-plugin-compression'
import legacy from '@vitejs/plugin-legacy'
import Inspect from 'vite-plugin-inspect'
import Components from 'unplugin-vue-components/vite'
import { AntDesignVueResolver } from 'unplugin-vue-components/resolvers'

export default defineConfig({
  plugins: [
    // 1. 框架插件（最优先）
    react(),  // 或 vue()
    vueJsx(), // Vue JSX 支持
    
    // 2. 组件自动导入插件
    Components({
      resolvers: [AntDesignVueResolver()]
    }),
    
    // 3. SVG 图标处理插件
    createSvgIconsPlugin({
      iconDirs: [path.resolve(process.cwd(), 'src/assets/icons')],
      symbolId: 'icon-[dir]-[name]'
    }),
    
    // 4. 开发工具插件（仅开发环境）
    Inspect(),  // 插件检查工具
    viteMockServe({
      mockPath: 'mock',
      enable: process.env.NODE_ENV === 'development'
    }),
    
    // 5. 构建优化插件（最后）
    legacy({
      targets: ['defaults', 'not IE 11']
    }),
    viteCompression({
      algorithm: 'gzip',
      ext: '.gz'
    })
  ]
})
```

#### 插件执行顺序说明：

**可以设置多个 `enforce: 'pre'` 或 `enforce: 'post'` 的插件！**

执行顺序如下：
```
1. enforce: 'pre' 的插件（按配置顺序执行）
   ↓
2. 普通插件（没有 enforce 选项，按配置顺序执行）
   ↓
3. Vite 内部插件
   ↓
4. enforce: 'post' 的插件（按配置顺序执行）
```

同一优先级的多个插件，按在 plugins 数组中的配置顺序执行（如：多个 pre 插件按配置顺序执行）。



### 4.2 插件的条件加载

可以根据环境条件加载插件：

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import { viteMockServe } from 'vite-plugin-mock'

export default defineConfig({
  plugins: [
    react(),
    // 只在开发环境加载
    process.env.NODE_ENV === 'development' && viteMockServe()
  ]
})
```
