# 4. 最佳实践和调试技巧

## 一、最佳实践

### 1.1 插件命名规范

- **命名格式**：使用 `vite-plugin-` 前缀
- **包名**：在 package.json 中使用 `vite-plugin-` 前缀
- **插件名称**：在插件对象的 `name` 属性中使用 `vite-plugin-` 前缀

### 1.2 插件结构最佳实践

1. **清晰的类型定义**：使用 TypeScript 定义插件选项
2. **合理的错误处理**：捕获并处理可能的错误
3. **详细的文档**：提供插件使用说明和示例
4. **单元测试**：为插件编写测试用例

### 1.3 性能优化

- **缓存机制**：对于重复的操作使用缓存
- **只处理必要的文件**：通过文件扩展名或路径过滤
- **避免同步操作**：使用异步 API 进行文件操作

## 二、调试技巧

### 2.1 基本调试

**使用** **`--debug`** **参数**：启用 Vite 的调试模式，输出详细的日志信息。

```bash
# 基本调试：显示所有调试信息
npm run dev -- --debug
```

**使用** **`console.log`**：在插件的关键位置添加日志。

```typescript
  return {
    name: 'my-plugin',
    transform(code, id) {
      console.log('Processing:', id)
      return code
    }
  }
}
```

### 2.2 高级调试

**使用环境变量控制调试输出**：通过 `DEBUG` 环境变量精确控制调试信息。

```bash
# 只显示插件相关的调试信息
DEBUG=vite:plugin npm run dev

# 只显示模块解析相关的调试信息
DEBUG=vite:resolve npm run dev

# 只显示构建相关的调试信息
DEBUG=vite:build npm run build

# 显示所有调试信息
DEBUG=* npm run dev
```

**使用 VS Code 断点调试**：

1. **创建调试配置**（`.vscode/launch.json`）：

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Debug Vite Plugin",
      "type": "node",
      "request": "launch",
      "runtimeExecutable": "npm",
      "runtimeArgs": ["run", "dev"],
      "cwd": "${workspaceFolder}",
      "env": {
        "DEBUG": "vite:plugin"
      },
      "console": "integratedTerminal",
      "internalConsoleOptions": "neverOpen"
    }
  ]
}
```

1. **在插件代码中设置断点**，然后启动调试。

**网络请求分析**：使用浏览器开发者工具分析网络请求。

1. 打开浏览器开发者工具（F12）
2. 切换到 "Network" 标签页
3. 刷新页面，查看 Vite 相关的网络请求
4. 分析模块加载和热更新请求

### 2.3 插件开发调试

**使用 debug 库**：创建可开关的调试日志。

1. **安装 debug 库**：

```bash
npm install debug -D
```

1. **在插件中使用**：

```typescript
import debug from 'debug'

// 创建调试实例
const log = debug('vite-plugin:my-plugin')

export function myPlugin() {
  return {
    name: 'vite-plugin-my-plugin',
    
    config(config, env) {
      log('Config received:', config)
      log('Environment:', env)
      return null
    },
    
    resolveId(source, importer) {
      log('Resolving:', source, 'from', importer)
      return null
    },
    
    transform(code, id) {
      log('Transforming:', id)
      return null
    }
  }
}
```

1. **运行调试**：

```bash
# 启用插件调试日志
DEBUG=vite-plugin:my-plugin npm run dev
```

**插件执行分析**：使用 `--debug` 查看插件钩子的执行顺序。

```bash
# 查看插件执行详情
DEBUG=vite:plugin npm run dev
```

**构建产物分析**：分析构建后的文件，检查插件的影响。

```bash
# 构建并分析产物
npm run build
# 查看 dist 目录下的文件
ls -la dist/
```

**常见调试场景**：

1. **模块解析问题**：使用 `DEBUG=vite:resolve` 查看模块解析过程
2. **插件执行顺序**：使用 `DEBUG=vite:plugin` 查看插件钩子执行顺序
3. **热更新问题**：使用 `DEBUG=vite:hmr` 查看热更新过程
4. **构建错误**：使用 `DEBUG=vite:build` 查看构建过程中的错误
   vite:resolve 解析结果: /Users/project/src/main.tsx +2ms
   vite:transform 转换文件: /src/main.tsx +0ms
   vite:transform 转换完成: /src/main.tsx +5ms

**在插件中添加 console.log 调试**：

```typescript
export function myPlugin() {
  return {
    name: 'my-plugin',
    transform(code, id) {
      console.log('Processing:', id)
      return code
    }
  }
}
```
